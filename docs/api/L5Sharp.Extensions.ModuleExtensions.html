Modules(Module)

Gets the child modules of this module component defined in the current L5X content.

public static IEnumerable<Module> Modules(this Module module)

Parameters

module Module

Returns

IEnumerable<Module>

A IEnumerable<T> of Module components that have the ParentModule property configured as the name of this module.

Remarks

The L5X structure serializes modules in a flat list with each element having properties (ParentModule, ParentModPortId) defining the parent/child IO tree relationship. It would be nice to navigate this hierarchy programatically, hence the reason for this extension method. Of course, this requires the module is attached to the L5X content. In-memory created modules will inherently return an empty collection, as there is no L5X structure to introspect.

Tags(Module)

Returns a collection of all non-null Tag objects for the current Module, including all config, input, and output tags.

public static IEnumerable<Tag> Tags(this Module module)

Parameters

module Module

Returns

IEnumerable<Tag>

An IEnumerable<T> containing the base tags for the Module.

Remarks

Since module tags are nested within different layers of complex types, it can be difficult to just get a single list of all module tags. This extension makes that easy by sifting through the object and returning a flat list containing all non-null config, input, and output tags defined for the Module component.
